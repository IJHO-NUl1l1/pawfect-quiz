# Pawfect Quiz — 나에게 맞는 강아지 찾기

재미있는 퀴즈로 사용자의 라이프스타일에 맞는 견종 Top 3를 추천하고, 결과가 예쁘고 공유하고 싶게 만드는 앱. 결과 공유를 통한 바이럴이 핵심 목표.

## 핵심 기능 (MVP 범위)

1. 10~12문항 퀴즈 (생활 패턴, 주거, 운동량, 성격 등)
2. 결과: Top 3 견종 + 귀여운 설명 + "이 강아지와의 상상 일상"
3. AI로 "나 + 추천 강아지" 합성 이미지 생성 (또는 예쁜 일러스트)
4. 결과 카드 이미지 다운로드 + 공유
5. (후순위) 친구 결과 비교, 같은 견종 키우는 사람들 피드

## 기술 스택

- **프레임워크**: Next.js (App Router) + TypeScript
- **스타일**: Tailwind CSS + shadcn/ui
- **애니메이션**: Framer Motion (퀴즈 문항 전환)
- **백엔드/DB**: Supabase (결과 저장, 매칭)
- **AI**: Vercel AI SDK 또는 이미지 생성 API
- **배포**: Vercel

## ⭐ 퀴즈 매칭 설계 — 이 프로젝트의 성패를 가르는 원칙

이 섹션이 가장 중요하다. 모든 데이터/퀴즈 관련 작업은 이 원칙을 따른다.

### 데이터 구조

견종 데이터를 특징(feature) 중심의 JSON으로 저장한다:

```jsonc
// 예시 개념 — 실제 스키마는 구현 시 확정
{
  "name": "Border Collie",
  "features": {
    "특징1": "값",
    "특징2": "값"
  }
}
```

- **각 특징 = 퀴즈 질문 하나.** 특징 세트가 정해지면 퀴즈 문항은 데이터에서 자동으로 도출된다.
- 사용자 답변과 견종 특징의 **일치 비율이 가장 높은 견종**이 결과가 된다 (점수 기반 정렬 → Top 3).

### 변별력 원칙 (최우선)

- 특징은 **견종 간 구분이 잘 되는(변별력 높은) 것들**로 구성해야 한다.
- 모든 견종이 비슷한 점수를 받아 퍼센트가 몰리면 퀴즈의 의미가 없다.
- 각 견종이 가진 **특이하고 구분되는 특징을 필두로** 특징 세트를 설계한다.
- 특징 후보를 고를 때는 실제 데이터에서 **분포/분산을 확인**해서 견종 간 값이 잘 퍼지는 특징을 우선 채택한다. (예: 거의 모든 견종이 3~4점인 특징은 탈락, 1~5점이 고르게 퍼지는 특징은 채택)

## 데이터 소스

하나만 고집하지 않고 **여러 소스를 병합**해서 사용한다. name/breed_id 기준으로 병합해 정제된 `breeds.json` 하나를 만드는 것이 목표.

| 소스 | 용도 | 비고 |
|---|---|---|
| [The Dog API](https://thedogapi.com) | 기본 정보 + 이미지 URL | 무료 API 키, ~200-300종(AKC 중심), temperament/weight/life_span 등 |
| [VildMedPap/dogbreeds](https://github.com/VildMedPap/dogbreeds) (DogTime 기반) | **숫자 평점 (매칭 로직 핵심)** | 349종, energy_level·apartment_friendly·good_with_children 등 1~5점 다수 |
| [open-dog-registry](https://registry.dog/api/v1) | 상세 구조화 데이터 | personalityTraits, coat, size 등. 오픈소스, 최신 |
| [tmfilho/akcdata](https://github.com/tmfilho/akcdata) | AKC 공식 277종 | 이미지 없음 → The Dog API 이미지와 조합 |
| Kaggle kkakey/dog_traits_AKC | 보조 trait 점수 (~200종) | 이미지 없음 |
| Dog CEO API | 이미지 보강용 | 메타데이터 거의 없음 |

**데이터 처리 방침**: 빌드 타임에 한 번 fetch/병합해서 `public/data/breeds.json`(또는 Supabase)에 저장. 런타임 외부 API 호출은 최소화.

## 진행 로드맵

1. **데이터 파이프라인**: 소스 fetch → 병합 → 변별력 분석 → 특징 세트 확정 → 정제된 `breeds.json` 생성 (스크립트로 재실행 가능하게)
2. **매칭 로직**: 특징 세트 기반 점수 계산 함수 + 단위 테스트 (변별력 검증 포함)
3. **퀴즈 UI**: 문항 흐름 + Framer Motion 전환 애니메이션
4. **결과 페이지**: Top 3 카드 + 견종 이미지 + 설명
5. **AI 이미지 생성**: 합성 이미지 또는 일러스트
6. **공유**: 결과 카드 이미지 다운로드 + 공유 링크 (Supabase 저장)
7. **후순위**: 친구 비교, 피드

각 단계는 이전 단계 산출물에 의존하므로 순서대로 진행한다. 특히 1~2단계(데이터 + 변별력)가 탄탄해야 이후가 전부 쉬워진다.

## 작업 규칙

- 사용자는 **단계별로 지시**한다. 지시받은 범위만 구현하고, 다음 단계를 임의로 진행하지 않는다.
- 사용자와의 소통은 한국어로 한다.
- API 키 등 시크릿은 `.env.local`에 두고 커밋하지 않는다.
- 데이터 생성/병합은 일회성 수작업이 아니라 `scripts/` 아래 재실행 가능한 스크립트로 만든다.
- 새 특징(질문)을 추가/제거할 때는 반드시 변별력 원칙(위 섹션)에 비추어 판단한다.
